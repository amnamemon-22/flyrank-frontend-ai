# Project Rules

## Stack
- React
- Vite
- JavaScript

## Coding Style
- Use functional components.
- Keep components small.
- Use meaningful variable names.
- Write clean and readable code.

## Project Rules Learned

1. Follow the existing project structure. Place reusable UI in `components`, page wrappers in `pages`, and styles in `styles`.

2. Reuse existing patterns before creating new ones. Match the project's coding style, naming conventions, and file organization.

3. Validate user input on the client side and provide clear inline error messages with accessible attributes such as `aria-invalid` and `aria-describedby`.

4. Review all AI-generated code before accepting it. Verify functionality, project consistency, and architecture before committing.
